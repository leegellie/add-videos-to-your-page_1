Networks

A network is a group of entities that can communicate, even though they are not directly conected.

By definition a network has more than 2 people and the data flow may have to go vias.

If the data map looks like a "net" in any way it could be a network.

To make a network you need:

  1. A way to encode and interpret the messages.
  2. A way to route the messages.
  3. Rules for deciding who gets to use the messages with what priorities.

Measuring Networks

The two main ways of mesuring a network are;

Latency:

The time it takes for a message to get from source to destination. This is measured in milliseconds, 1000 milliseconds = 1 second.

Bandwidth:

The amount of information that can be transmitted through the network during a set amount of time. This is measured in Million Bits Per Second, Mbps.

Bits:

A bit is the smallest part of data you can use on a computer. The is Boolean and there is yes or no. These are actually a 0 and 1.

Objects, Classes & Instances

Objects are bits of data that have a location in the memory of the program we are running or working with it.

Objects can be variables, structured data or functions. Objects can be made and several others are pre-defined. The can be referenced, called upon and changed in your code.

In the case of "Classes" the object is the specific instance of that class.

Classes are a new thing we learned about that have some exciting and very, very useful properties.

When we started we were just using functions. This was great and with changing variable you could make the functions do slightly different things... this was limiting and for more extensive programming would require a LOT of code writing.

Classes are essentially "preset" functions. This means that there can be a whole set of functions assigned to help with a certain programming procedure.

Classes also have a great feature of beign called and assigned a new INSTANCE. In this case each instance is given its own version of that class and can be used as THAT instance forever on. Unlike giving a function an alias. That class will no longer change if a another instance is made for it. Only specific instruction to that instance will effect that instance.

You could have as many instances of that class as you want and all will operate independantly.

To create a new instance you only need to define a variable calling the instance. That variable will then be that instance of the class.

Methods & Instance Variables

Methods are functions in the class that can be called when using that class. They are written by defining them under the class.

Methods are called by Class.method().

Instance Variable are the different variables that can be used by a class. This makes it so that you can call various variables again and again but can be different with each instance. They, like just about everything inside a class code, are started with self which tells the class the variable is to be used with the specific instance of that class.

So if you had a .Building class you could have Building height, Building depth, building width, floor size, floor numbers, etc. as instance variables and every time you called a new instance of that class you could pass it the variable for that instance and could make a different "Building" for each.

Constructors

Although this is not really gone over in the course, the Constructor is used in a class to create an object. It readies that new object, with passed input if needed, for use.

It has to open a space in the memory to run the Class. It is started with __init__ to initialize the space in the memory.

In our case this was also defining the main 'function' of our class.

Constant Variables

Constant variable are variables that have been set and are unlikely to be changed. The Google Python Style Guide suggests making that variable in ALL CAPS. This help you know off hand which things will be the same every time you use them.

Class Variables

When you set a variable in the base of your class it becomes valid and callable for all instances of that class.

It becomes a property of that class that can be set and pulled as needed. For example; If you set the class variable of Line to be 3cm long you could use it every time you call that class. You can also set it specifically for the instance you are using.

Pre-Existing Class Variables

All Python classes come with some pre-existing variables. These are useful in working with and debugging classes.

Inheritance

Inheritance is where attributes, functions, etc., can be taken from other classes to be used in another class and/or into your final code.

Methods & Method Overriding

A method is a function in a class. Sometimes you will have a class with a method, or function, defined in an inherited class. You may want it to show something else. In that case you just need to define the same method in the newer class.

Difference Between OOP and other programming

Object oriented programming utalizes functions, objects, that are set or can be defined and then pulled whenever you need them in your program. This, as mentioned earlier, avoids repetition and can make your programming change based on what is defined in different variables in your code.

Unlike a procedural programming language, like HTML, you can have the code dynamically modify based on other factors. HTML would have to, in sequence, define each part of the output and it is then set there. With OOP, like Python, you can only create the code that is needed, modify that code based on different variables and many other nifty things.

One of the most important parts of OOP is the "Classes" it utalizes. These, like blueprints, have the basic structure of a procedure (wich we usually call a "method" in OOP. But unlike a procedure written in procedural programming you can use the same class over and over and just call that class with new variables, or the same, to make it do what you want.

If you used HTML and CSS to create a document, you have no way to change what the code is without some kind of programming that will rewrite the code. With OOP you can do that very easily.

For example. You can call a class like MakeHTMLp which will for each instance have a creator that will write a line starting with "<p>" and ending in "</p>" and it will fill with a variable and can have a variable set to assign it a CSS class, etc. You do not need to re-write the function every time you want to make the program spit out HTML code. Just call the class with the applicable variables.

So with this you could have your program read several different files and write all the different blocks with one class.
